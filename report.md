1. 实验概述
本实验旨在使用 LoRA (Low-Rank Adaptation) 技术，对 Google 的 Gemma-3-270M 小语言模型进行微调，使其能根据详细的文本描述生成有效的 SVG 徽标代码。

实验的核心是：
设计一个程序化的奖励（Reward）函数来量化 SVG 的质量
以该函数为优化目标进行 LoRA 微调
观察微调模型是否比基座模型有提升
核心发现：微调模型在验证集上的平均奖励分数从 0.0588 提升至 0.2676，有效 SVG 生成率从 11.76% 提升至 35.29%，验证了 LoRA 微调在小模型上的有效性。

2. 奖励函数设计
2.1 设计理念
我设计了一个多维度、加权评分的奖励函数，从七个维度评估 SVG 徽标的质量：

维度	权重	评分理由
结构完整性	20%	SVG 必须有正确的根元素和 xmlns 等必要属性，这是有效性的基础
标签闭合性	20%	未闭合的标签会导致渲染失败，是代码严谨性的核心指标
提示词覆盖度	15%	衡量模型是否遵循了用户指令，体现了模型的"理解能力"
颜色有效性	15%	确保 fill 和 stroke 属性使用合法颜色值，保证渲染正确
元素多样性	15%	好的徽标应由多种图形元素组成，避免过于简单或重复
坐标合法性	10%	防止坐标越界（-100 到 1000），避免元素飞出画布
大小合理性	5%	防止过短（<50字符）的退化输出或过长（>10000字符）的冗余代码
2.2 评分逻辑
每个维度独立评分（0-1分）
加权求和得到总分
硬性惩罚：如果结构无效、标签未闭合或坐标越界，总分减半
最终分数归一化到 0-1 范围

python
# 核心评分逻辑示例
total_score = sum(scores[key] * weights[key] for key in weights)
if not valid or not all_closed or not within_bounds:
    total_score *= 0.5  # 硬性惩罚
total_score = max(0.0, min(1.0, total_score))
2.3 设计理由
我选择这种设计是因为：
有效性优先：SVG 首先必须是"可渲染的"，否则一切免谈
质量导向：在有效的基础上，颜色、元素多样性等决定了一个徽标的"好坏"
任务契合：提示词覆盖度直接反映了模型是否按照指令执行
鲁棒性：大小检查防止模型"钻空子"生成退化的无效输出

5. 分析
5.1 为什么有提升？
LoRA 微调有效引导了模型：模型学会了生成 SVG 的基本结构（<svg> 根元素、必要的属性、闭合标签）。
Token 准确率提升：从 64.8% 到 71.1%，说明模型更倾向于输出与训练数据相似的 token 序列。
部分样本达到可用水平：最高分 0.85 的样本说明模型已经能够生成结构完整、颜色有效、标签闭合的 SVG。

5.2 为什么提升有限？
模型太小（270M）：Gemma-3-270M 比 Sonnet 小几百倍，学习和记忆复杂 SVG 模式的能力有限。
任务复杂度高：SVG 是结构化的 XML 语言，需要精确的语法、标签匹配和属性组合，对模型要求很高。
数据量不足：只有 219 条训练样本，模型难以学习到 SVG 生成的完整分布。
奖励函数的"天花板"：当前奖励函数主要检查"有效性"（结构、闭合、颜色等），对"美学质量"的评估有限。

5.3 Goodhart 效应观察
有趣的是，我观察到微调模型虽然学会了生成更完整的 SVG 结构，但在提示词覆盖度上的提升并不明显（从 11.76% 到 35.29%）。这说明：
模型在"钻空子"：它学会了生成"形式上有效"但内容与提示词无关的 SVG
代理指标的局限性：我的奖励函数侧重于结构和语法，对内容的相关性评估不足
如果继续优化，模型可能会生成大量"有效但跑题"的 SVG，这就是典型的 Goodhart 效应——代理分数上去了，真实质量却没跟上。

6. 结论
6.1 主要发现
LoRA 微调有效：在 Gemma-3-270M 上，LoRA 微调显著提升了 SVG 生成的质量（平均分 +355%，有效率 +200%）。
小模型的潜力与局限：270M 模型能够学会生成结构完整的 SVG，但复杂度和美观度仍有限。
奖励函数设计的重要性：良好的奖励函数能有效引导模型学习，但也需要注意避免 Goodhart 效应。

6.2 实验局限
训练数据量较小（219 条），限制了模型的学习效果
奖励函数主要关注"有效性"，对"美学"和"创意"的评估不足
验证集规模有限（17 条），统计显著性可能不足

6.3 未来改进方向
  数据增强：通过旋转、缩放、颜色变换等方式扩充训练数据 ;奖励函数迭代：引入更细粒度的美学评估（如布局平衡、配色和谐度） ;模型升级：尝试更大的模型（如 1B-4B 级别）以获得更好的生成质量 强化学习：使用 RLHF 或 PPO 直接优化奖励函数
